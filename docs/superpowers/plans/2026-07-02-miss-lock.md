# MISS Button Double-Click Lock Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a double-click gesture on the MISS button that locks it armed across multiple shots (instead of auto-disarming after one), staying locked until double-clicked again.

**Architecture:** One new module-level state variable (`missLock`), a click-vs-double-click disambiguation timer in the MISS button's click handler, an updated shot-recording condition, updated render markup, and one new CSS rule. No new files.

**Tech Stack:** Vanilla JS (`app.js`), plain CSS (`styles.css`), no build step.

## Global Constraints

- No other button gains double-click behavior — only `#btn-miss`.
- Existing single-click one-shot arm/disarm behavior is unchanged when `missLock` is `false`.
- Once `missLock` is `true`, a single click on MISS is a no-op — only another double-click turns it off.
- `missLock` resets to `false` at the same two places `missArm` already resets: inside `openHistoryGame` (`app.js:1011`) and inside `startGame` (`app.js:1060`).
- `undo` is not touched — `missLock` (like the existing `missArm`) is not part of the undo-tracked game log.

- [ ] **Step 5: Replace the MISS click handler in `wireGame`**

In `app.js`, find this block (around line 1424-1428):

```js
  $('btn-miss') &&
    ($('btn-miss').onclick = () => {
      missArm = !missArm;
      render();
    });
```

Replace with:

```js
  let missClickTimer = null;
  $('btn-miss') &&
    ($('btn-miss').onclick = () => {
      if (missClickTimer) {
        clearTimeout(missClickTimer);
        missClickTimer = null;
        missLock = !missLock;
        missArm = false;
        render();
        return;
      }
      missClickTimer = setTimeout(() => {
        missClickTimer = null;
        if (!missLock) {
          missArm = !missArm;
          render();
        }
      }, 300);
    });
```

`missClickTimer` is a local variable declared inside `wireGame()` (like the function's other closures) — it is re-created fresh each time `wireGame()` runs (i.e. each render), which is correct: it only needs to track pending clicks between renders of the same button instance.

There is no automated test for this interaction (no pure-function coverage applies — matches the precedent set by prior click/interaction-only changes). Since this sandbox has no browser available, ask the user to verify in the actual app:

1. Single-click MISS (not locked): after a brief pause it turns red (armed), same as before this change; tapping a shot (with a player selected) records a miss and MISS turns back off.
